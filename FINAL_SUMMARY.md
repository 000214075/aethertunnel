## 📋 发布步骤（用户执行）

### 第 1 步：创建 GitHub 仓库（10 分钟）

1. 访问 https://github.com/new
2. 仓库名称：`aethertunnel`
3. 可见性：Public
4. 不初始化（已有代码）
5. 点击 "Create repository"

### 第 2 步：本地 Git 配置（5 分钟）

```bash
cd /workspace/projects/workspace/aethertunnel

# 初始化 Git（如果还没初始化）
git init

# 添加所有文件
git add .

# 首次提交
git commit -m "Initial commit: AetherTunnel v0.1.0 MVP"

# 添加远程仓库
git remote add origin https://github.com/your-username/aethertunnel.git
```

### 第 3 步：推送代码到 GitHub（10 分钟）

```bash
# 创建主分支
git checkout -b main

# 推送代码
git push -u origin main
```

### 第 4 步：创建 Release 标签（5 分钟）

```bash
# 创建标签
git tag -a v0.1.0 -m "Release v0.1.0 - MVP"

# 推送标签
git push origin v0.1.0
```

### 第 5 步：GitHub Actions 自动构建和发布（30-60 分钟）

推送标签后会自动触发 GitHub Actions：
1. 运行测试
2. 编译所有 14 个平台的二进制文件
3. 压缩二进制文件
4. 生成 SHA256 校验和
5. 自动创建 v0.1.0 Release
6. 上传所有平台的二进制文件

### 第 6 步：验证 Release（10 分钟）

1. 访问 GitHub 仓库的 Releases 页面
2. 检查 v0.1.0 Release 是否存在
3. 下载几个平台的二进制文件
4. 验证文件完整性（SHA256 校验）

### 第 7 步：发布通知（1 周）
